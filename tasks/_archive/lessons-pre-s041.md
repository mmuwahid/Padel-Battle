## Patterns That Work
- [2026-04-01] Component extraction with view-local state (S019) — moving useState + functions that belong exclusively to one view produces self-contained components with zero cross-view breakage risk. AdminDashboard's 5 useState + 4 functions are fully isolated. Key: identify state that is NEVER read by other views.
- [2026-04-01] esbuild syntax check on all modified files before starting dev server — catches import/JSX issues in seconds without a full Vite build. Run after every multi-file refactor.
- [2026-03-30] Full UI walkthrough with preview tools catches real user-facing bugs that code review misses — BF-27 (no sign-out on LeagueGate) and BF-28 (invite visible to non-admins) were both invisible in code review but immediately obvious when using the app as a real user.
- [2026-03-29] LeagueGate internal toast system (S016) — giving a child component its own toast state that delegates to parent's showToast when available avoids complex state lifting. Pattern: `const _toast=(msg,type)=>{if(showToast)showToast(msg,type);else{setToast({msg,type});setTimeout(()=>setToast(null),3000);}}`
- [2026-04-01] SECURITY DEFINER RPC functions for Edge Function DB access (S020) — completely eliminates RLS/permission issues in Edge Functions. Use anon key + RPC calls instead of service role key + direct table queries. The RPC functions bypass RLS at the DB level, which is more reliable than client-side key handling.
- [2026-05-02] Native iOS-correct numeric input combo: `type="text" inputMode="numeric" pattern="[0-9]*"` — Why: `type="number"` silently breaks `setSelectionRange` and cursor placement on iOS Safari. The text-mode trio keeps the numeric keypad on mobile while restoring full cursor / backspace / digit-by-digit editing. Use for any digit-only field where editing fluency matters (scores, OTPs, PINs, quantities). Pair with empty-string-allowing controlled value `value={n===0?"":String(n)}` and a digit-strip onChange (`r.replace(/[^0-9]/g,"")`) so the field can be cleared mid-edit without storing non-numeric state. Removed `onFocus={e=>e.target.select()}` at the same time — it forces select-all on every tap and defeats the cursor fix.
- [2026-05-02] Advisor as scope guard before substantive UX work (S040) — Why: advisor caught a "extract a stepper component" scope creep before any code was written. CLAUDE.md "Mockup first" / "No design changes without approval" rules would have been violated otherwise. Calling advisor before the FIRST edit on any non-trivial UX task is cheaper than a rollback and surfaces project-rule violations the planner agent would miss.
- [2026-04-01] Testing Edge Function end-to-end from preview dev server (S020) — catches real deployment issues (JWT verification, RLS, missing GRANTs) that local testing misses. Always test with the actual Supabase project, not mocks.
- [2026-04-01] JSONB `responses` column for per-player tracking (S021) — using `{player_id: "accepted"|"declined"}` in a JSONB column is cleaner than a separate junction table for small datasets (4 players). Easy to check `allPlayerIds.every(p => responses[p] === "accepted")` for auto-confirm logic.
- [2026-04-01] Testing on actual production URL via Chrome browser tools (S022) — caught the 400 errors and blank screen that were invisible on the dev server (dev server had a different auth session). Always verify the PRODUCTION deploy, not just the dev server.
- [2026-04-01] Plain object for React Context value (S022) — safer than useMemo when value contains derived state. Minor re-render cost is negligible vs crash risk from dependency cascades. Pattern: `const ctx = { ...values };` not `const ctx = useMemo(() => ({...}), [deps])`.
- [2026-04-01] Checking Supabase network requests in browser DevTools (S022) — HTTP 400 status on specific queries immediately reveals which `.select()` columns are wrong. `read_network_requests` with `urlPattern: "supabase"` is the fastest diagnostic for data loading issues.
- [2026-04-01] Triple-layer review (S024: structured code review + adversarial Codex audit + deep explore) — each pass catches different classes of bugs. Code reviewer found runtime crashes (undefined vars, wrong columns). Codex found RLS/auth/race conditions. Deep explore found performance anti-patterns with line numbers. Use all three for comprehensive audits.
- [2026-04-01] Adversarial security audit should test the DATABASE LAYER, not just the UI — hiding a button in React doesn't prevent direct Supabase calls. RLS policies are the real security boundary. Any audit must check what a user can do via `supabase.from().update()` directly, not just what the UI exposes.
- [2026-04-01] Supabase CLI `db query --linked -f` for SQL migrations (S025) — write migration to `docs/S###-description.sql`, execute via CLI, verify with `pg_policies` query. Faster than Dashboard, scriptable, leaves local documentation.
- [2026-04-01] SECURITY DEFINER RPC for cross-boundary queries (S025 `lookup_league_by_invite`) — allows non-members to look up a league by invite code without granting SELECT on the leagues table. Returns minimal data (id, name only). Pattern for restricted access: create a SECURITY DEFINER function returning only needed fields.
- [2026-04-02] SECURITY DEFINER + FOR UPDATE for atomic multi-user state (S026) — eliminates read-modify-write races entirely. Client sends intent ("player X accepts challenge Y"), server handles mutation atomically with row lock. Scales to any concurrent state problem.
- [2026-04-02] Debounced realtime with useRef timer (S026) — cleaner than lodash.debounce, no extra dependency, auto-cleanup on unmount. Pattern: `const ref=useRef(null); const fn=useCallback(()=>{clearTimeout(ref.current);ref.current=setTimeout(action,500);},[])`.

## Critical Rules (Always Enforce)
1. **Mockup-first workflow:** HTML mockup → user review → finalize → implement. Never skip.
2. **Nav bar is LOCKED.** Do not touch tab IDs, labels, icons, or layout.
3. **No design changes without explicit approval.** If not explicitly decided, don't change it.
4. **Placeholder data only in mockups.** No invented names, leagues, or stats.
5. **GitHub deploy process:** Git CLI clone → copy → commit → push. Vercel auto-deploys.
6. **Version control planning docs.** Always V1, V2, V3 naming. Never overwrite originals. Keep all versions in planning/.
7. **Never regenerate user documents.** Edit originals via XML. Add addendums, not replacements.
8. **Match existing app styling in mockups.** Copy CSS from most recent approved mockup. No new design patterns without approval.
9. **Safe area insets:** Always use `env(safe-area-inset-*)` for fixed-position elements near screen edges on mobile.
10. **Verify before pushing fixes.** Run dev server (preview_start), check console errors, confirm the affected screen renders. Build passing ≠ feature working. Never push without testing.
11. **When adding inline state-driven UI (ternaries replacing prompt/confirm), always declare the useState first.** Grep to confirm: `grep -n "useState.*varName"` before committing.
12. **CRITICAL — Verify DB columns before writing Supabase `.select()` with specific columns.** Run `SELECT column_name FROM information_schema.columns WHERE table_name='X'` for EVERY table being optimized. Never trust code-level research for DB schema. A wrong column name causes HTTP 400 that gets silently swallowed, making data appear deleted. (S022 incident — 2.5 hours of broken production.)
13. **Never use useMemo for React Context values that include derived state.** Use a plain object. useMemo dependency cascades (A depends on B which depends on C) cause infinite re-render loops that are invisible in dev but crash production minified builds.
14. **Never place hooks (useState, useMemo, useEffect, useCallback) after conditional returns.** React requires hooks to execute in the same order every render. After adding early returns, grep for hook calls below any `return` statement. Violation causes silent blank screen crash with no useful error message. (S025 BF-32 — player profile blank screen.)
