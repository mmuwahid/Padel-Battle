## Key Decisions

1. **AppContent stays in App.jsx for now** — it's the state container with 15-20+ state variables and deeply coupled data fetching. Extracting it further requires introducing context or a state management library, which is a separate task.
2. **AuthGate and LeagueGate import supabase directly** — they are top-level wrappers that manage auth state independently, not receiving supabase as a prop.
3. **All other components receive supabase as a prop** — no direct imports, keeping them testable and decoupled.
4. **Safe Restructure Protocol** — user provided a 5-phase protocol. Backup created first, one extraction at a time, verify after each, revert on errors.

## Lessons Learned

- **Vite HMR caches aggressively** — after restructuring files, a full server restart is needed to clear stale module references. HMR errors on "already declared" identifiers are false positives from cache.
- **Extract bottom-up** — start with zero-dependency utilities (theme, helpers, data), then small components, then large ones. This minimizes risk at each step.
- **Backup before refactor** — the `_wip/backup-pre-refactor-2026-03-27/` saved us from stress. Always create before touching structure.
