## Key Decisions
- **Approach B (State + JSX extraction)** chosen over Approach A (JSX-only). Moving view-local state with the JSX reduces AppContent's useState count, which was the root cause of S013/S014 regressions.
- **Avatar state stays in AppContent** — used by header, sidebar, and profile view. Moving it would require lifting it back up or adding context (deferred to A-04).
- **Notification state stays in AppContent** — push subscription logic is shared with push functions used by match logging.
- **editDisplayName initialization moved to component mount** — was previously set by header avatar button onClick. Now SettingsView initializes from user.user_metadata on mount, which is cleaner.
- **A-04 (React Context) and A-09 (SELECT *) deferred** — user chose focused extraction-only scope for lower risk.

## Lessons Learned

### Validated Patterns
- [2026-04-01] Extracting components with their view-local state is the right granularity — AdminDashboard's 5 useState and 4 functions are fully self-contained, zero risk of cross-component breakage.
